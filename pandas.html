üêº Pandas Introduction

Pandas is a Python library used for working with data sets. It has functions for analyzing, cleaning, exploring, and manipulating data. The name "Pandas" has a reference to both "Panel Data", and "Python Data Analysis" and was created by Wes McKinney in 2008.

üìä Why Use Pandas?

Pandas allows us to analyze big data and make conclusions based on statistical theories. Pandas can clean messy data sets, and make them readable and relevant. Relevant data is very important in data science.

Data Science: is a branch of computer science where we study how to store, use and analyze data for deriving information from it.

üîç What Can Pandas Do?

Pandas gives you answers about the data. Like:

  • Is there a correlation between two or more columns?
  • What is the average value?
  • What is the max value?
  • What is the min value?

Pandas is also able to delete rows that are not relevant, or contain wrong values, like empty or NULL values. This is called cleaning the data.

‚öôÔ∏è Installation

If you have Python and PIP already installed on a system, then installation of Pandas is very easy. Install it using this command:

C:\Users\Your Name>pip install pandas

If this command fails, then use a Python distribution that already has Pandas installed like Anaconda, Spyder etc.

üì• Import Pandas

Once Pandas is installed, import it in your applications by adding the import keyword. Create an alias with the as keyword while importing:

import pandas as pd

Now the Pandas package can be referred to as pd instead of pandas.

Module 1: Getting Started with Pandas

Task 1: Your First Pandas DataFrame

Create a simple dataset using a dictionary and turn it into a Pandas DataFrame ‚Äî the most used data structure in Pandas.

import pandas as pd # Create a dictionary with our data mydataset = { 'cars': ["BMW", "Volvo", "Ford"], 'passings': [3, 7, 2] } # Convert the dictionary into a DataFrame myvar = pd.DataFrame(mydataset) # Display the DataFrame print(myvar)
cars passings
0 BMW 3
1 Volvo 7
2 Ford 2
üìù Line-by-Line Explanation:
  • import pandas as pd - We import the Pandas library and give it the short alias pd so we can use it conveniently throughout our code.
  • mydataset = { ... } - We create a Python dictionary where each key becomes a column name and each value (a list) becomes the column data.
  • pd.DataFrame(mydataset) - The DataFrame() function converts our dictionary into a table-like structure with rows and columns, just like an Excel spreadsheet!
  • print(myvar) - Displays the DataFrame. Notice how Pandas automatically adds index numbers (0, 1, 2) on the left side.

Module 2: Pandas Series

üìã What is a Series?

A Pandas Series is like a column in a table. It is a one-dimensional array holding data of any type.

Think of a Series as a single vertical column of data ‚Äî it can hold numbers, strings, or any Python object. A DataFrame is essentially a collection of Series.

Task 2: Creating a Simple Series

Create a Pandas Series from a basic Python list.

import pandas as pd # Create a Python list a = [1, 7, 2] # Convert the list into a Pandas Series myvar = pd.Series(a) # Display the Series print(myvar)
0 1
1 7
2 2
dtype: int64
üìù Line-by-Line Explanation:
  • a = [1, 7, 2] - We create a simple Python list containing three integer values.
  • pd.Series(a) - The Series() function converts our list into a Pandas Series. Each value gets assigned an automatic index (0, 1, 2).
  • print(myvar) - Displays the Series. Notice the output shows two columns: the index on the left and the values on the right, plus the dtype (data type) at the bottom.

Task 3: Accessing Series Values by Index

Use the index number to retrieve a specific value from a Series ‚Äî just like with lists!

import pandas as pd a = [1, 7, 2] myvar = pd.Series(a) # Access the first value using index 0 print(myvar[0])
1
üìù Line-by-Line Explanation:
  • myvar[0] - If nothing else is specified, the values are labeled with their index number. First value has index 0, second value has index 1, and so on. Here we access the first value which is 1.

Task 4: Creating Custom Labels

Use the index argument to name your own labels instead of using default numeric indexes.

import pandas as pd a = [1, 7, 2] # Create a Series with custom labels myvar = pd.Series(a, index = ["x", "y", "z"]) print(myvar)
x 1
y 7
z 2
dtype: int64
üìù Line-by-Line Explanation:
  • index = ["x", "y", "z"] - The index argument lets you replace the default numeric labels (0, 1, 2) with your own custom labels. Now the value 1 has the label "x", 7 has "y", and 2 has "z".

Task 5: Accessing Items by Custom Label

When you have created custom labels, you can access an item by referring to the label instead of the index number.

import pandas as pd a = [1, 7, 2] myvar = pd.Series(a, index = ["x", "y", "z"]) # Access the value with label "y" print(myvar["y"])
7
üìù Line-by-Line Explanation:
  • myvar["y"] - Instead of using a number like myvar[1], we can now use the string label "y" to access the value 7. This makes our code much more readable and meaningful!

Task 6: Key/Value Objects as Series (Dictionary)

You can also use a key/value object, like a dictionary, when creating a Series. The keys of the dictionary automatically become the labels.

import pandas as pd # Create a dictionary with calorie data calories = {"day1": 420, "day2": 380, "day3": 390} # Convert dictionary to a Series myvar = pd.Series(calories) print(myvar)
day1 420
day2 380
day3 390
dtype: int64
üìù Line-by-Line Explanation:
  • calories = {"day1": 420, ...} - We create a Python dictionary where each key represents a day and each value represents the calorie count for that day.
  • pd.Series(calories) - When you pass a dictionary to Series(), the keys automatically become the labels (index), and the values become the data. No need to specify the index argument manually!

Task 7: Selecting Specific Items from a Dictionary

To select only some items from the dictionary, use the index argument and specify only the items you want to include.

import pandas as pd calories = {"day1": 420, "day2": 380, "day3": 390} # Create a Series using only "day1" and "day2" myvar = pd.Series(calories, index = ["day1", "day2"]) print(myvar)
day1 420
day2 380
dtype: int64
üìù Line-by-Line Explanation:
  • index = ["day1", "day2"] - Even though our dictionary has 3 items (day1, day2, day3), the index argument acts as a filter. It tells Pandas: "Only include these specific keys!" So day3 is completely excluded from the resulting Series.

Module 3: Pandas DataFrames ‚Äî Deep Dive

üìä What is a DataFrame?

A Pandas DataFrame is a 2-dimensional data structure, like a 2-dimensional array, or a table with rows and columns.

Think of it like this:

  • A Series = one single column (1D)
  • A DataFrame = the whole table with many columns (2D)

It's essentially the Python version of an Excel spreadsheet or a SQL database table. Every time you work with real-world data in Pandas, you'll be using DataFrames!

Task 8: Creating a Simple DataFrame

Build a full DataFrame by providing a dictionary where each key is a column and each value is a list of data.

import pandas as pd # Create a dictionary with two columns of data data = { "calories": [420, 380, 390], "duration": [50, 40, 45] } # Load data into a DataFrame object df = pd.DataFrame(data) print(df)
calories duration
0 420 50
1 380 40
2 390 45
üìù Line-by-Line Explanation:
  • data = { "calories": [...], "duration": [...] } - We create a dictionary with two keys. Each key becomes a column header, and the corresponding list becomes the column data. Both lists must have the same length!
  • df = pd.DataFrame(data) - The DataFrame() constructor takes our dictionary and builds a beautiful 2D table. By convention, we name our DataFrame variable df ‚Äî this is a standard practice you'll see everywhere!
  • print(df) - Displays the complete DataFrame. Notice how it looks like a proper spreadsheet with column names at the top and row indexes (0, 1, 2) on the left!

Module 4: Locating Rows in a DataFrame

üîé Locate Row

As you can see from the DataFrames above, the structure is like a table with rows and columns. But how do we pull out specific rows?

Pandas uses the loc attribute to return one or more specified row(s). This is one of the most powerful tools in your Pandas toolkit!

Task 9: Locating a Single Row

Use df.loc[] to extract a single row from the DataFrame by its index number.

import pandas as pd data = { "calories": [420, 380, 390], "duration": [50, 40, 45] } df = pd.DataFrame(data) # Refer to the row index: print(df.loc[0])
calories 420
duration 50
Name: 0, dtype: int64
üìù Line-by-Line Explanation:
  • df.loc[0] - The loc attribute is like a GPS for your data! We give it the row index 0 (the first row), and it returns all the data from that row.
  • Name: 0, dtype: int64 - Notice the output format: it shows each column name on the left and its value on the right. This is actually a Pandas Series ‚Äî when you grab a single row, Pandas returns it as a Series, not a DataFrame!

Task 10: Locating Multiple Rows

Use a list of indexes inside df.loc[[]] to extract multiple rows at once.

import pandas as pd data = { "calories": [420, 380, 390], "duration": [50, 40, 45] } df = pd.DataFrame(data) # Use a list of indexes: print(df.loc[[0, 1]])
calories duration
0 420 50
1 380 40
üìù Line-by-Line Explanation:
  • df.loc[[0, 1]] - Notice the double brackets [[ ]]! When you pass a list of indexes, Pandas grabs multiple rows and returns them as a proper DataFrame (not a Series). Here we asked for row 0 and row 1.
  • ü§î Key Difference: Using df.loc[0] (single brackets) returns a Series. Using df.loc[[0, 1]] (double brackets with a list) returns a DataFrame. This matters when you use the result in further operations!

Module 5: Named Indexes

üè∑Ô∏è Named Indexes

With the index argument, you can name your own indexes instead of using the default numbers (0, 1, 2...).

This is extremely useful in real-world data! Instead of remembering "row 0 = Monday's data", you can just name the row "Monday".

Task 11: Adding Named Indexes to a DataFrame

Add a list of names to give each row a meaningful name instead of a boring number.

import pandas as pd data = { "calories": [420, 380, 390], "duration": [50, 40, 45] } # Add named indexes using the index argument df = pd.DataFrame(data, index = ["day1", "day2", "day3"]) print(df)
calories duration
day1 420 50
day2 380 40
day3 390 45
üìù Line-by-Line Explanation:
  • index = ["day1", "day2", "day3"] - We pass a list of custom names to the index argument. These names replace the default numeric indexes (0, 1, 2) with meaningful labels.
  • print(df) - Look at the output! Instead of boring numbers on the left, we now see day1, day2, day3. This makes the data immediately understandable at a glance!

Task 12: Locating Rows by Named Index

Use the named index in the loc attribute to return a specific row by its name instead of a number.

import pandas as pd data = { "calories": [420, 380, 390], "duration": [50, 40, 45] } df = pd.DataFrame(data, index = ["day1", "day2", "day3"]) # Refer to the named index: print(df.loc["day2"])
calories 380
duration 40
Name: day2, dtype: int64
üìù Line-by-Line Explanation:
  • df.loc["day2"] - Instead of remembering that "day2" is at index 1, we simply use the name directly! Pandas finds the row labeled "day2" and returns all its data.
  • Name: day2, dtype: int64 - The result is a Series showing calories = 380 and duration = 40 for day2. Notice the "Name" field at the bottom now shows "day2" instead of a number ‚Äî because we used named indexes!

Module 6: Loading Files into DataFrames

üìÅ Load Files Into a DataFrame

In the real world, your data won't be typed into dictionaries by hand. It will live in files ‚Äî CSV files, Excel spreadsheets, JSON files, and databases.

Pandas can load all of these into a DataFrame with a single line of code! This is where Pandas truly becomes a superpower for data analysis.

Task 13: Loading a CSV File

Load a comma-separated file (CSV file) into a DataFrame using pd.read_csv().

import pandas as pd # Load a CSV file into a DataFrame df = pd.read_csv('data.csv') # Display the entire DataFrame print(df)
üìù Line-by-Line Explanation:
  • pd.read_csv('data.csv') - This single line does ALL the heavy lifting! Pandas opens the file named "data.csv", reads every row and column, and automatically converts it into a full DataFrame. The first row of the CSV file typically becomes the column headers.
  • print(df) - Displays the loaded data. If the file has hundreds of rows, Pandas will smartly show only the first and last 5 rows with ... in between, so your screen doesn't overflow!
  • üí° Tip: You can also load other file formats: pd.read_excel('data.xlsx') for Excel files, pd.read_json('data.json') for JSON files, and even pd.read_sql() for databases!

ü§ñ Why JSON Matters in Machine Learning

JSON plays a critical role in the Machine Learning pipeline:

  • API Responses ‚Äî When you fetch data from APIs (e.g., Twitter, Google, weather services), the data comes back as JSON. ML models need this data to train on real-world information.
  • Model Configuration ‚Äî Hyperparameters and model settings are often stored in JSON files (e.g., config.json in Hugging Face models).

Module 10: Data Cleaning

üßπ Data Cleaning

Data cleaning means fixing bad data in your data set.

Bad data could be:

  • ‚ùå Empty cells ‚Äî missing values that leave gaps in your data
  • ‚ùå Data in wrong format ‚Äî dates stored as text, numbers as strings
  • ‚ùå Wrong data ‚Äî typos or incorrect values that don't make sense
  • ‚ùå Duplicates ‚Äî the same row appearing multiple times

In this module you will learn how to deal with all of them.

ü§ñ Why Data Cleaning is CRITICAL in Machine Learning

In the world of Machine Learning, there's a famous saying:

"Garbage In, Garbage Out"

If you feed dirty data to an ML model, you'll get bad predictions. Here's why data cleaning is essential:

  • Missing Values ‚Äî Most ML algorithms cannot handle NaN/null values. Your model will crash or produce wrong results if you don't handle them first.
  • Wrong Format ‚Äî If a "Date" column contains mixed formats (some as strings, some as dates), your model won't understand the time patterns in the data.
  • Outliers & Wrong Data ‚Äî An incorrect value like 450 minutes of exercise (instead of 45) will skew your model's understanding and lead to inaccurate predictions.
  • Duplicates ‚Äî Duplicate rows make the model think certain patterns are more common than they actually are, creating bias in your predictions.

Module 11: Cleaning Empty Cells

üï≥Ô∏è Empty Cells

Empty cells can potentially give you a wrong result when you analyze data.

There are two main strategies to deal with empty cells:

  • Remove the rows that contain empty cells
  • Replace the empty cells with a calculated value (mean, median, or mode)

üóëÔ∏è Remove Rows

One way to deal with empty cells is to remove rows that contain empty cells.

This is usually OK, since data sets can be very big, and removing a few rows will not have a big impact on the result.

Module 13: Fixing Wrong Data

‚ùå Wrong Data

"Wrong data" does not have to be "empty cells" or "wrong format" ‚Äî it can just be wrong, like if someone registered "199" instead of "1.99".

‚úèÔ∏è Replacing Values

One way to fix wrong values is to replace them with something else.

  • 0.9 ‚Üí Very good relationship ‚Äî if you increase one value, the other will probably increase too

This is a very strong correlation! It means: the longer you work out, the more calories you burn üî•. And the other way around ‚Äî if you burned a lot of calories, you probably had a long workout. This makes perfect sense!

‚ùå Bad Correlation: Duration vs Maxpulse = 0.009403

This is almost zero ‚Äî a terrible correlation. It means: you cannot predict someone's max pulse just by looking at how long they worked out. A 30-minute sprint might give you a higher max pulse than a 60-minute walk!

ü§ñ Why Correlation Matters in Machine Learning

Correlation is one of the first things Data Scientists check before building an ML model:

  • Feature Selection ‚Äî Columns with high correlation to the target (what you're predicting) are great features for your model. Low correlation columns might be useless and can be dropped.
  • Multicollinearity ‚Äî If two input features are highly correlated with each other (like Pulse and Maxpulse at 0.79), they provide overlapping information. Some models perform better if you remove one of them.
  • Data Understanding ‚Äî Correlation helps you understand your data BEFORE training. It's like reading a map before starting a journey! üó∫Ô∏è

Import Pyplot from Matplotlib and visualize the entire DataFrame as a line chart.

import pandas as pd import matplotlib.pyplot as plt df = pd.read_csv('data.csv') # Plot all numeric columns as line charts df.plot() # Display the plot plt.show()

üîµ Scatter Plot

A scatter plot shows the relationship between two columns as dots on a graph.

Specify that you want a scatter plot with: kind = 'scatter'

A scatter plot needs an x-axis and a y-axis.

üìä Histogram

A histogram shows the frequency of values in a column ‚Äî how many times each value range appears.

Specify that you want a histogram with: kind = 'hist'

A histogram only needs one column.

For example: "How many workouts lasted between 50 and 60 minutes?"

Task 43: Create a Histogram

Create a histogram showing the frequency distribution of workout durations.

import pandas as pd import matplotlib.pyplot as plt df = pd.read_csv('data.csv') # Create a histogram of the Duration column df["Duration"].plot(kind = 'hist') plt.show()
üìù Line-by-Line Explanation:
  • df["Duration"].plot(kind = 'hist') - We select just one column ("Duration") and create a histogram. Each bar represents a range of values (called a "bin"), and the height shows how many workouts fall in that range.
  • üí° When to use Histograms: Histograms are perfect for understanding the distribution of your data. They answer questions like: "What's the most common value?", "Are there outliers?", "Is the data normally distributed?"
  • ü§ñ ML Use Case: In Machine Learning, histograms help you understand:
    • Whether your data is normally distributed (bell-shaped) ‚Äî many models assume this!
    • Whether there are outliers that need cleaning
    • Whether you need to transform your data (e.g., log transformation for skewed data)
