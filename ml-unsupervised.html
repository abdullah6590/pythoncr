🔍 Unsupervised Learning

What happens when data has no labels? The machine finds hidden patterns, groups, and structures all by itself — like a detective solving a mystery without witnesses!

🎓 What is Unsupervised Learning?

Imagine you're given a bag of 1000 mixed candies — no labels, no categories. You start sorting them by color, shape, and size on your own. Nobody told you how to group them — you just discovered the patterns yourself. That's Unsupervised Learning!

📖 The Definition

Unsupervised Learning is a type of machine learning where the model is trained using unlabeled data. There are no correct answers provided — the algorithm must discover hidden patterns, structures, and relationships on its own.

Unlike supervised learning where we say "this is a cat, this is a dog," here we just throw all the data in and say: "Find something interesting!"

🔬 The Math Behind It

The goal is to discover hidden structure in data:

Find patterns in X (no y!)
X Input data only — features without any labels
No y No target variable — the model discovers structure itself
f The learned function that groups, reduces, or finds patterns

⚔️ Supervised vs Unsupervised — Key Difference

▸ Input: 1000 photos → Groups: ???

🔑 Key Characteristics

🚫

No Labels Required

The data has no correct answers — the model must figure out the structure on its own.

🕵️

Self-Discovery

Learning is unsupervised — like a detective finding clues without witnesses.

🧩

Pattern & Structure Tasks

Used to find groups, reduce dimensions, and detect anomalies in data.

📦 Types of Unsupervised Learning

Unsupervised learning comes in three main flavors. Each solves a different kind of problem!

🎯 The Three Types

🫧

Clustering

Group similar data points together. Like sorting fruits into baskets without knowing their names!

Examples: K-Means, DBSCAN, Hierarchical

📉

Dimensionality Reduction

Simplify complex data by reducing features while keeping the important information.

Examples: PCA, t-SNE, UMAP

🔗

Association Rule Learning

Find relationships between items. "People who buy X also buy Y!"

Examples: Apriori, FP-Growth

Step 1: K-Means Clustering — Group Customers! 🫧

K-Means is the most popular clustering algorithm. It divides data into K groups by finding cluster centers. Like sorting students into study groups based on their skills!

🎯 How K-Means Works

1. Pick K random centers. 2. Assign each point to nearest center. 3. Move centers to the average of their group. 4. Repeat until centers stop moving!

import numpy as np import matplotlib.pyplot as plt from sklearn.cluster import KMeans from sklearn.datasets import make_blobs # 🎨 Generate sample customer data (spending habits) X, y_true = make_blobs(n_samples=300, centers=4, cluster_std=0.8, random_state=42) # Apply K-Means with 4 clusters kmeans = KMeans(n_clusters=4, random_state=42, n_init=10) labels = kmeans.fit_predict(X) centers = kmeans.cluster_centers_ # 📊 Visualize the clusters! plt.figure(figsize=(10, 6)) colors = ['#38bdf8', '#a78bfa', '#34d399', '#fbbf24'] for i in range(4): mask = labels == i plt.scatter(X[mask, 0], X[mask, 1], c=colors[i], s=50, alpha=0.7, label=f'Cluster {i+1}') # Mark cluster centers plt.scatter(centers[:, 0], centers[:, 1], c='red', s=200, marker='X', edgecolors='white', linewidths=2, label='Centers', zorder=5) plt.title('🫧 K-Means Clustering — Customer Segments', fontsize=14) plt.xlabel('Feature 1 (e.g., Annual Income)') plt.ylabel('Feature 2 (e.g., Spending Score)') plt.legend(); plt.grid(alpha=0.2); plt.tight_layout(); plt.show() print(f"✅ Found {len(set(labels))} customer segments!") print(f"📊 Cluster sizes: {[sum(labels == i) for i in range(4)]}")

Step 2: Finding the Best K — Elbow Method 📐

How do you know how many clusters to use? The Elbow Method! Plot the "inertia" (total distance from points to their cluster center) for different K values. The "elbow" in the curve = optimal K!

# 📐 Elbow Method — find the optimal number of clusters inertias = [] K_range = range(1, 11) for k in K_range: km = KMeans(n_clusters=k, random_state=42, n_init=10) km.fit(X) inertias.append(km.inertia_) plt.figure(figsize=(8, 5)) plt.plot(K_range, inertias, marker='o', color='#38bdf8', linewidth=2, markersize=8) plt.axvline(x=4, color='#ef4444', linestyle='--', alpha=0.7, label='Elbow at K=4') plt.xlabel('Number of Clusters (K)', fontsize=12) plt.ylabel('Inertia (Within-cluster distance)', fontsize=12) plt.title('📐 Elbow Method — Finding Optimal K', fontsize=14) plt.legend(); plt.grid(alpha=0.3); plt.tight_layout(); plt.show() print("💡 The 'elbow' is at K=4 — that's our optimal number of clusters!")

Step 3: DBSCAN — Density-Based Clustering 🔬

K-Means needs you to specify K upfront. But DBSCAN finds clusters automatically based on density! It can even find oddly-shaped clusters and detect outliers (noise points). Think of it like finding groups of people at a party — dense groups are clusters, isolated people are outliers!

from sklearn.cluster import DBSCAN from sklearn.datasets import make_moons # 🌙 Generate crescent-shaped data — K-Means can't handle this! X_moons, _ = make_moons(n_samples=300, noise=0.08, random_state=42) # DBSCAN finds clusters by density dbscan = DBSCAN(eps=0.2, min_samples=5) labels_db = dbscan.fit_predict(X_moons) # Compare K-Means vs DBSCAN fig, axes = plt.subplots(1, 2, figsize=(14, 5)) # K-Means (fails with crescent shapes!) km = KMeans(n_clusters=2, random_state=42, n_init=10) labels_km = km.fit_predict(X_moons) axes[0].scatter(X_moons[:, 0], X_moons[:, 1], c=[colors[l] for l in labels_km], s=40) axes[0].set_title('❌ K-Means (Fails!)', fontsize=13) # DBSCAN (handles it perfectly!) scatter_colors = ['#38bdf8' if l == 0 else '#a78bfa' if l == 1 else '#ef4444' for l in labels_db] axes[1].scatter(X_moons[:, 0], X_moons[:, 1], c=scatter_colors, s=40) axes[1].set_title('✅ DBSCAN (Perfect!)', fontsize=13) plt.suptitle('🔬 K-Means vs DBSCAN on Crescent Data', fontsize=15) plt.tight_layout(); plt.show() n_clusters = len(set(labels_db)) - (1 if -1 in labels_db else 0) n_noise = sum(labels_db == -1) print(f"✅ DBSCAN found {n_clusters} clusters and {n_noise} noise points!")

Step 4: Hierarchical Clustering — Dendrogram Tree 🌳

Hierarchical Clustering builds a tree of clusters called a dendrogram. It starts with each point as its own cluster, then keeps merging the closest pairs. You can "cut" the tree at any level to get different numbers of clusters! Like a family tree for your data 🌳

from scipy.cluster.hierarchy import dendrogram, linkage from sklearn.cluster import AgglomerativeClustering # 🌳 Generate sample data X_hier, _ = make_blobs(n_samples=50, centers=3, cluster_std=1.0, random_state=42) # Create linkage matrix for dendrogram linkage_matrix = linkage(X_hier, method='ward') # Plot dendrogram fig, axes = plt.subplots(1, 2, figsize=(15, 5)) # Dendrogram view dendrogram(linkage_matrix, ax=axes[0], color_threshold=7, above_threshold_color='#636e72') axes[0].set_title('🌳 Dendrogram (Cut at height ≈ 7)', fontsize=13) axes[0].set_xlabel('Sample Index'); axes[0].set_ylabel('Distance') axes[0].axhline(y=7, color='#ef4444', linestyle='--', label='Cut line (3 clusters)') axes[0].legend() # Cluster visualization agg = AgglomerativeClustering(n_clusters=3) labels_agg = agg.fit_predict(X_hier) scatter_colors = ['#38bdf8', '#a78bfa', '#34d399'] for i in range(3): mask = labels_agg == i axes[1].scatter(X_hier[mask, 0], X_hier[mask, 1], c=scatter_colors[i], s=80, label=f'Cluster {i+1}') axes[1].set_title('📊 Resulting 3 Clusters', fontsize=13) axes[1].legend(); axes[1].grid(alpha=0.2) plt.suptitle('🌳 Hierarchical Clustering with Dendrogram', fontsize=15) plt.tight_layout(); plt.show()

Step 5: PCA — Dimensionality Reduction 📉

Principal Component Analysis (PCA) is like summarizing a 500-page book into a 1-page summary — you keep the most important information while removing the noise. It reduces the number of features while preserving maximum variance!

🎯 Why PCA?

Imagine a dataset with 100 features. Hard to visualize, right? PCA squashes it down to 2-3 dimensions so you can see the patterns. It's also great for speeding up other ML algorithms!

from sklearn.decomposition import PCA from sklearn.datasets import load_iris from sklearn.preprocessing import StandardScaler # 🌸 Load Iris dataset (4 features) iris = load_iris() X_iris = StandardScaler().fit_transform(iris.data) print(f"📋 Original shape: {X_iris.shape} (150 samples × 4 features)") # Reduce from 4 dimensions to 2! pca = PCA(n_components=2) X_reduced = pca.fit_transform(X_iris) print(f"📉 Reduced shape: {X_reduced.shape} (150 samples × 2 features)") print(f"📊 Variance retained: {sum(pca.explained_variance_ratio_):.1%}") # 📊 Visualize 4D data in 2D! plt.figure(figsize=(10, 6)) colors = ['#38bdf8', '#a78bfa', '#34d399'] for i, name in enumerate(iris.target_names): mask = iris.target == i plt.scatter(X_reduced[mask, 0], X_reduced[mask, 1], c=colors[i], s=60, label=name, alpha=0.8) plt.xlabel(f'PC1 ({pca.explained_variance_ratio_[0]:.1%} variance)') plt.ylabel(f'PC2 ({pca.explained_variance_ratio_[1]:.1%} variance)') plt.title('📉 PCA: 4D Iris Data → 2D Visualization', fontsize=14) plt.legend(); plt.grid(alpha=0.2); plt.tight_layout(); plt.show() print("💡 Even in 2D, the species are clearly separable!")

Step 6: Measuring Clustering Quality — Silhouette Score 📊

Since there are no labels, how do we know if our clusters are good? The Silhouette Score measures how similar points are to their own cluster vs. neighboring clusters. Score = 1 is perfect, 0 is random, -1 is wrong clustering.

from sklearn.metrics import silhouette_score # Generate data X_eval, _ = make_blobs(n_samples=300, centers=4, cluster_std=0.8, random_state=42) # Test different K values K_range = range(2, 8) silhouette_scores = [] print("📊 Silhouette Scores for different K:") for k in K_range: km = KMeans(n_clusters=k, random_state=42, n_init=10) labels = km.fit_predict(X_eval) score = silhouette_score(X_eval, labels) silhouette_scores.append(score) emoji = "🏆" if score > 0.7 else "✅" if score > 0.5 else "⚠️" print(f" {emoji} K={k}: Silhouette Score = {score:.4f}") # Plot Silhouette Scores plt.figure(figsize=(8, 5)) plt.bar(K_range, silhouette_scores, color=['#38bdf8' if s == max(silhouette_scores) else '#636e72' for s in silhouette_scores]) plt.xlabel('Number of Clusters (K)', fontsize=12) plt.ylabel('Silhouette Score', fontsize=12) plt.title('📊 Silhouette Score — Finding Best K', fontsize=14) plt.xticks(list(K_range)); plt.grid(alpha=0.2, axis='y') plt.tight_layout(); plt.show() best_k = list(K_range)[silhouette_scores.index(max(silhouette_scores))] print(f"\n🏆 Best K = {best_k} (Silhouette Score = {max(silhouette_scores):.4f})")
